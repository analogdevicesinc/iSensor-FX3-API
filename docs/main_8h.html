◆ AdiAppErrorHandler()

void AdiAppErrorHandler ( CyU3PReturnStatus_t  status)

This function handles critical errors generated by the ADI application.

At the moment this function handles errors by printing an error message to the UART console and looping indefinitely. In the future it might be prudent to do something more intelligent here based on the specific error condition.

◆ AdiAppStart()

void AdiAppStart ( void  )

This function sets up the necessary resources to start the ADI application.

It configures all GPIO and timers used by the firmware, as well as the USB endpoints, DMA controller, and SPI hardware.

◆ AdiAppStop()

void AdiAppStop ( void  )

This function is called to shut down the application.
